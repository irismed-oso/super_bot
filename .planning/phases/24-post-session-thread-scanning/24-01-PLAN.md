**2. Module-level setup:**
- Import anthropic (with lazy import pattern matching memory_store.py -- wrap in try/except so bot runs without it)
- Import structlog for logging
- Import bot.memory_store for storage and dedup
- Use `anthropic.AsyncAnthropic()` client (reads ANTHROPIC_API_KEY from env automatically)
- Use model `claude-sonnet-4-20250514` for extraction (fast, cheap, sufficient for extraction)

**3. `async def scan_and_store(client, channel: str, thread_ts: str, user_id: str, task_summary: str)` -- the main entry point called from result_cb:**
- This is the fire-and-forget function. Wrap entire body in try/except that logs warnings but never raises (background task must not crash).
- Step 1: Store task history immediately via `_store_task_history(task_summary, user_id, channel)`
- Step 2: Fetch thread messages via `client.conversations_replies(channel=channel, ts=thread_ts, limit=100)`
- Step 3: Filter to human messages only -- skip messages where `msg.get("bot_id")` is set OR `msg.get("subtype")` is set (SCAN-04: never extract from bot's own messages)
- Step 4: If no human messages remain (edge case), return early
- Step 5: Build extraction input by joining human messages as numbered lines: `"{i}. {msg['text']}"`
- Step 6: Call `_extract_memories(text_block)` to get extracted items
- Step 7: For each extracted item, deduplicate via `memory_store.search(item, limit=3)` -- if any result has content similarity > 80% (simple substring check: if the extracted item is a substring of existing content or vice versa, skip it), then skip. Otherwise call `memory_store.store(content=item, category=memory_store.categorize(item), source_user=user_id, source_channel=channel)`
- Step 8: Log count of memories stored

**4. `async def _extract_memories(human_text: str) -> list[str]` -- the Claude extraction call:**
- Create the extraction prompt (system + user). The system prompt must enforce conservative extraction (SCAN-03):

```
SYSTEM: You extract memorable facts and directives from Slack conversations.

Rules:
- ONLY extract explicit directives: statements containing "always", "never", "the rule is", "make sure to", "don't forget", "from now on", or similar imperative language
- ONLY extract stated facts: concrete information like names, dates, configurations, procedures, preferences
- DO NOT extract: questions, speculative statements ("maybe", "I think", "could be"), bot instructions, pleasantries, status updates about current tasks, temporary context
- DO NOT extract anything that is just describing what the bot should do RIGHT NOW for this specific task
- Return each extracted memory on its own line, one per line
- If nothing is worth remembering, return exactly: NONE
- Keep each memory concise (1-2 sentences max)
- Do not number the lines
```

- User message: `"Extract memorable information from this conversation:\n\n{human_text}"`
- Call `anthropic_client.messages.create(model="claude-sonnet-4-20250514", max_tokens=500, system=SYSTEM_PROMPT, messages=[{"role": "user", "content": user_msg}])`
- Parse response: split by newlines, strip whitespace, filter empty lines
- If response is "NONE" or empty, return empty list
- Return list of extracted strings

**5. `async def _store_task_history(summary: str, user_id: str, channel: str)` -- task history auto-capture (SCAN-05):**
- Store a one-line summary with category="history" via `memory_store.store(content=summary, category="history", source_user=user_id, source_channel=channel)`
- The summary is passed in from result_cb (derived from clean_text + result subtype)

**Important constraints:**
- Use `anthropic.AsyncAnthropic()` (async client, compatible with existing event loop)
- The entire scan_and_store function must be wrapped in try/except -- failures must NEVER propagate since this runs as a detached background task
- Log all operations at info/warning level via structlog
- Do NOT extract from bot messages (check for bot_id or subtype fields in Slack message objects)
