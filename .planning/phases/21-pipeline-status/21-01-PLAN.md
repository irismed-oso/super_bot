  <action>
Create `bot/pipeline_status.py` following the `bot/log_tools.py` CLI pattern (argparse, async main, `python -m bot.pipeline_status` entry point).

**Prefect API query:**
- POST to `{PREFECT_API}/flow_runs/filter` with a JSON body filtering by `start_time` (after the computed window start) and sorting by `START_TIME_DESC`
- Import `PREFECT_API` and `PREFECT_AUTH` from `bot.prefect_api` -- do NOT duplicate credentials
- Use httpx async client with the same auth tuple pattern as prefect_api.py
- Limit results to 100 runs max to avoid huge responses

**CLI interface:**
- `python -m bot.pipeline_status` -- default last 24 hours
- `python -m bot.pipeline_status --hours 48` -- last N hours
- `python -m bot.pipeline_status --since "2026-03-25"` -- since a specific date/time (ISO format)
- The agent will translate Nicole's natural language ("today", "this week", "03.25") into the appropriate `--hours` or `--since` flag

**Formatting (printed to stdout for agent to capture):**
- Group runs by state type: FAILED/CRASHED first, then RUNNING/PENDING, then COMPLETED
- Failed section: emphasized header, each run shows flow name, state message (error detail), start time, run ID
- Running section: each run shows flow name, start time, run ID
- Completed section: show individual runs up to ~10, then summarize remainder as "and N more completed"
- Include run durations where available (end_time - start_time)
- Summary line at top: "Pipeline Status (last 24h): X completed, Y failed, Z running"
- Keep total output under 2500 chars for Slack safety (truncate completed list if needed)

**Error handling:**
- If Prefect API is unreachable, print a clear error message (not a traceback)
- If zero runs found, print "No flow runs found in the specified time window"

**Key implementation details:**
- Use `datetime.utcnow()` for time calculations (Prefect API uses UTC)
- Parse state from `run["state"]["type"]` -- valid values: COMPLETED, FAILED, RUNNING, PENDING, SCHEDULED, CANCELLING, CANCELLED, CRASHED
- Group FAILED and CRASHED together as "Failed"
- Group RUNNING, PENDING, SCHEDULED together as "Running"
- Group CANCELLING, CANCELLED with "Other" if present
- Flow name from `run["name"]` is the run name; flow name from `run.get("flow_id")` needs a separate lookup -- instead use the `name` field which Prefect auto-generates, and include the deployment/flow name if available in the response fields
