  <action>
Create bot/digest_changelog.py with an async function `build_changelog_section(target_date: date) -> str` that:

1. **Query activity log for git entries:**
   - Call `activity_log.read_day_by_type(target_date, "git_commit")` to get logged commits
   - Call `activity_log.read_day_by_type(target_date, "git_pr")` to get logged PRs

2. **Cross-check git log for missed commits (DGCL-03):**
   - Build a list of repo paths to scan: the primary repo from `os.environ.get("MIC_TRANSFORMER_CWD", "/home/bot/mic_transformer")` plus all paths in `config.ADDITIONAL_REPOS`
   - For each repo path that exists on disk, run `git log --all --since="{target_date}T00:00:00" --until="{target_date+1day}T00:00:00" --format="%H|%s" --name-only` via `asyncio.create_subprocess_exec`
   - Parse the output using the same `%H|%s` + name-only format that git_activity.py already uses (extract into a helper or reparse inline)
   - For each commit found in git log, check if its hash already exists in the activity log entries. If not, add it to the results as a "missed" commit (this handles crashes, DGCL-03)
   - Derive repo name from `os.path.basename(repo_path)`

3. **Group by repository (DGCL-02):**
   - Build a dict keyed by repo name, value is `{"commits": [...], "prs": [...]}`
   - Sort repos alphabetically

4. **Format the changelog section (DGCL-01):**
   - If no commits and no PRs across all repos, return empty string (changelog section omitted)
   - Otherwise return a formatted block like:
     ```
     *Changelog*

     *mic_transformer* (3 commits, 1 PR)
     - `abc1234` fix: update parser logic
     - `def5678` feat: add new endpoint
     - `ghi9012` test: add parser tests
     - PR: title of PR — https://url

     *oso-fe-gsnap* (1 commit)
     - `jkl3456` fix: correct template
     ```
   - Commit hashes truncated to 7 chars
   - Commit messages truncated to 80 chars
   - PR entries show title and URL
   - Cap at 15 commits per repo (with "...and N more" overflow)

Use structlog for logging. Handle subprocess failures gracefully (log warning, continue with activity log data only — cross-check is best-effort). Import config at module level for ADDITIONAL_REPOS.
