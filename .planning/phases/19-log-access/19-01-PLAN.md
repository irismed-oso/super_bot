<task type="auto">
  <name>Task 1: Create log_tools module with journald and Prefect log retrieval</name>
  <files>bot/log_tools.py, tests/test_log_tools.py</files>
  <action>
Create bot/log_tools.py with the following functions:

1. **async fetch_journald_logs(service, lines=100, grep=None, since=None)** -> str
   - Runs `journalctl -u {service} -n {lines} --no-pager --output=short-iso` via asyncio.create_subprocess_exec
   - If `grep` is provided, add `--grep={grep}` (case-insensitive via `--case-sensitive=no`)
   - If `since` is provided (e.g. "1h", "30m", "2h"), add `--since="{since} ago"`
   - Use service aliases from deploy_state.REPO_CONFIG to resolve names (e.g. "superbot" -> systemd unit "superbot", "mic" -> resolve via REPO_CONFIG service field)
   - Returns parsed, truncated, scrubbed output via format_log_output()

2. **async fetch_prefect_logs(run_id_or_name)** -> str
   - Uses httpx to call Prefect API: GET /flow_runs/{id}/logs or POST /flow_runs/filter to find by name
   - Use same PREFECT_API and PREFECT_AUTH from bot/prefect_api.py (import constants)
   - Returns parsed, truncated output via format_log_output()

3. **parse_structlog_line(line)** -> str
   - Attempts to parse a line as JSON (structlog format)
   - If JSON with "timestamp", "level", "event" keys: returns "timestamp level event" (any extra keys with useful info like error= appended)
   - If not JSON: returns the line as-is
   - This handles mixed output (some lines JSON, some plain)

4. **scrub_secrets(text)** -> str
   - Regex patterns for common secret formats: API keys (sk-*, xoxb-*, xoxp-*), passwords in URLs, AWS keys, bearer tokens
   - Replaces with "[REDACTED]"
   - Keep patterns conservative -- better to miss a secret than mangle real data

5. **format_log_output(raw_lines, max_chars=2800)** -> str
   - Parse each line through parse_structlog_line()
   - Scrub the full output via scrub_secrets()
   - If total chars > max_chars, truncate from the beginning (keep most recent lines), prepend "... (showing last N of M lines)"
   - Return the formatted string

6. **resolve_service_name(alias)** -> str | None
   - Resolves user-facing aliases ("superbot", "sb", "mic") to systemd unit names
   - Uses REPO_CONFIG from deploy_state.py -- the "service" field has the systemd unit name
   - Returns None if not recognized

Create tests/test_log_tools.py:
- Test parse_structlog_line() with JSON input and plain text input
- Test scrub_secrets() with various secret patterns (sk-key, xoxb-token, password in URL, bearer token)
- Test format_log_output() truncation behavior (over limit triggers truncation message)
- Test resolve_service_name() with known aliases and unknown input
