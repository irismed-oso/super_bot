**Regex patterns (anchored prefix to avoid collisions):**

```python
# Memory commands -- BEFORE deploy commands to avoid collision
_REMEMBER_RE = re.compile(r"^\s*remember\b\s+(.+)", re.IGNORECASE | re.DOTALL)
_RECALL_RE = re.compile(
    r"^\s*(?:recall|what\s+do\s+you\s+know\s+about|what\s+do\s+you\s+remember\s+about)\b\s+(.+)",
    re.IGNORECASE | re.DOTALL,
)
_FORGET_RE = re.compile(r"^\s*forget\b\s+(.+)", re.IGNORECASE | re.DOTALL)
_LIST_MEMORIES_RE = re.compile(
    r"^\s*list\s+memories(?:\s+(rules?|facts?|history|preferences?))?",
    re.IGNORECASE,
)
```

**Handlers:**

1. `_handle_remember(text, **kwargs)`:
   - Extract content from match group 1
   - Auto-categorize via `memory_store.categorize(content)`
   - Get `slack_context` from kwargs to extract `user_id` and `channel`
   - Call `memory_store.store(content, category, source_user, source_channel)`
   - Return: "Remembered as *{category}*: _{content[:100]}_"
   - On failure: "Failed to store memory. The memory system may be unavailable."

2. `_handle_recall(text, **kwargs)`:
   - Extract query from match group 1
   - Call `memory_store.search(query, limit=10)`
   - If no results: "No memories found matching '{query}'."
   - Format results as numbered list:
     ```
     Found {N} memories matching "{query}":
     1. [{category}] {content} -- stored by <@{source_user}> on {created_at}
     2. ...
     ```
   - Include memory ID in parentheses at end of each line for reference: `(id: {id})`

3. `_handle_forget(text, **kwargs)`:
   - Extract query from match group 1
   - First check if query is a numeric ID (direct delete): `memory_store.get_by_id(int(query))`
   - If numeric and found: deactivate immediately, return "Forgot memory #{id}: _{content[:80]}_"
   - If not numeric: search via `memory_store.search(query, limit=5)`
   - If 0 results: "No memories found matching '{query}'."
   - If 1 result: deactivate it, return "Forgot: _{content[:80]}_"
   - If multiple results: list them with IDs and say "Multiple matches found. Use `forget {id}` to remove a specific one:" followed by numbered list with IDs

4. `_handle_list_memories(text, **kwargs)`:
   - Extract optional category filter from match group 1
   - Normalize category: "rules" -> "rule", "facts" -> "fact", "preferences" -> "preference"
   - Call `memory_store.list_all(category=category_filter, limit=50)`
   - If no memories: "No memories stored yet. Use `remember [text]` to add one."
   - Group by category and format:
     ```
     *Rules* (2)
     - {content} (id: {id})
     - {content} (id: {id})

     *Facts* (3)
     - ...
     ```
   - If category filter applied, show only that category
