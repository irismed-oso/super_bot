  <action>
Create `bot/heartbeat.py` with a `Heartbeat` class that manages:

**Shared mutable state:**
- `turn_count: int` (starts at 0, incremented by on_message callback)
- `last_activity: str` (starts as "Starting up...", updated by milestone detection)
- `_started_at: float` (time.time() when heartbeat starts)
- `_progress_msg: dict | None` (the Slack message dict with ts/channel to edit)
- `_client` (Slack async client reference)
- `_task: asyncio.Task | None` (the running timer task)
- `_stopped: bool` (flag to prevent ghost edits after stop)

**Methods:**
- `start(client, progress_msg: dict)` -- stores references, sets `_started_at = time.time()`, creates `asyncio.create_task(self._loop())`. If progress_msg is None, log warning and skip (no message to edit).
- `stop()` -- sets `_stopped = True`, cancels `_task` if running, awaits cancellation. Idempotent (safe to call multiple times).
- `_loop()` -- async method: sleep 60s (first heartbeat), then loop sleeping 180s (3-min intervals). On each tick, if not `_stopped`, call `_tick()`. Wrap entire loop in try/except asyncio.CancelledError for clean shutdown.
- `_tick()` -- builds message text using format: `:hourglass: Still working... {last_activity} | Turn {turn_count}/{MAX_TURNS} | {elapsed}` where elapsed uses `progress._format_elapsed()`. Calls `client.chat_update(channel=progress_msg["channel"], ts=progress_msg["ts"], text=text)`. Log each tick with structlog. Catch and log Slack API errors (do not crash the timer).

Import MAX_TURNS from bot.agent. Import `_format_elapsed` from bot.progress (rename it to `format_elapsed` by removing the underscore prefix to make it a public function).

**Update `bot/progress.py`:**
- Rename `_format_elapsed` to `format_elapsed` (remove underscore). Update the one internal call site in `post_result` to use `format_elapsed`.
- Update `make_on_message()` to accept an optional `heartbeat: Heartbeat | None = None` parameter. When a milestone is detected AND heartbeat is not None, also set `heartbeat.last_activity = milestone` (the short label like "Reading files...", "Making changes...", etc). When any AssistantMessage arrives (regardless of milestone), if heartbeat is not None, increment `heartbeat.turn_count += 1`.
- When milestone fires AND heartbeat is provided, also build the full heartbeat-format string and use it as the text for the `chat_update` call (instead of just the bare milestone). This way milestone updates show the same format as heartbeat ticks: `:hourglass: Still working... {milestone} | Turn {turn_count}/{MAX_TURNS} | {elapsed}`. This requires heartbeat to expose a `format_message()` method that builds the string, or the milestone code builds it directly using heartbeat's state.

Per user decision: milestone fires update the message immediately with the new activity label in heartbeat format. The heartbeat timer continues independently and will update elapsed time on its next tick.
