<task type="auto">
  <name>Task 1: Create reusable deploy script</name>
  <files>scripts/deploy.sh</files>
  <action>
Create `scripts/deploy.sh` -- a reusable deploy script with NO hardcoded version numbers or milestone-specific logic.

The script must:

1. **Accept optional flags:**
   - `--branch BRANCH` (default: current branch, pushes before deploying)
   - `--skip-push` (skip the git push step, useful when already pushed)
   - `--skip-deps` (skip pip install step for fast restarts)

2. **Step sequence** (modeled on existing deploy_v1.1.sh patterns):
   - Step 1: Push current branch to origin (unless --skip-push)
   - Step 2: SSH to VM, pull latest code on the bot's super_bot repo
     - Use `gcloud compute ssh bot@superbot-vm --zone=us-west1-a`
     - Pull with: `sudo -u bot bash -c 'cd /home/bot/super_bot && git pull origin main'`
   - Step 3: Install Python dependencies (unless --skip-deps)
     - `sudo -u bot bash -c 'cd /home/bot/super_bot && source .venv/bin/activate && uv pip install -r requirements.txt'`
   - Step 4: Restart systemd service
     - `sudo systemctl restart superbot`
     - `sleep 3`
   - Step 5: Health check
     - Check `systemctl is-active superbot` -- must return "active"
     - Check `journalctl -u superbot -n 20 --no-pager` for crash indicators (look for "ERROR" or "Traceback" in last 20 lines)
     - Print service status with `systemctl status superbot --no-pager`

3. **Exit behavior:**
   - Exit 0 with green "DEPLOY SUCCESS" message if service is active and no crash indicators
   - Exit 1 with red "DEPLOY FAILED" message if service is not active or crash detected
   - Print a final summary line: service status, uptime info

4. **Variables at top of file (not hardcoded in steps):**
   ```bash
   VM="superbot-vm"
   ZONE="us-west1-a"
   BOT_USER="bot"
   SERVICE="superbot"
   REPO_DIR="/home/bot/super_bot"
   ```

5. **Use `set -euo pipefail`** and include a usage/help section triggered by `--help`.

Do NOT include any version-specific steps (like adding env vars, cloning repos, deploying skills). Those belong in milestone-specific runbook sections of DEPLOY.md.

Reference `scripts/restart_superbot.sh` for the gcloud SSH heredoc pattern and `scripts/deploy_v1.1.sh` for the step structure.
