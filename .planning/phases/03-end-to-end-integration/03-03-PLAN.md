**`async def post_started(client, channel: str, thread_ts: str, task_text: str) -> None`**
Posts a brief "started" message to the thread. Format:
```
Working on it: {first 80 chars of task_text}...
```
(Truncate task_text to 80 chars + "..." only if longer than 80. If 80 or less, no ellipsis.)
Use `client.chat_postMessage(channel=channel, thread_ts=thread_ts, text=msg)`.
Wrap in try/except — log but don't raise on Slack API errors.

**`def make_on_message(client, channel: str, thread_ts: str)`**
Returns an async callback `on_message_cb(message: AssistantMessage)` suitable for passing as `on_message=` to `run_agent_with_timeout()`.

Inside the callback:
- Extract tool names: `tools = [b.name for b in message.content if isinstance(b, ToolUseBlock)]`
- Extract bash command strings: concatenate `b.input.get("command", "")` for all Bash ToolUseBlocks
- Determine milestone using this priority order:
  1. If any tool in `_WRITE_TOOLS` → "Making changes..."
  2. Elif "glab mr" in bash_cmd or "mr create" in bash_cmd → "Creating MR..."
  3. Elif "pytest" in bash_cmd → "Running tests..."
  4. Elif "git commit" in bash_cmd or "git push" in bash_cmd → "Committing changes..."
  5. Elif any tool in `_READ_TOOLS` → "Reading files..."
  6. Else → `None` (don't post)
- Track `last_milestone` via closure variable (initialized to `None`)
- Only post if `milestone is not None` AND `milestone != last_milestone`
- Update `last_milestone = milestone` before posting
- Post via `client.chat_postMessage(channel=channel, thread_ts=thread_ts, text=milestone)`
- Wrap posting in try/except — log warning but don't raise

Return the callback function.

**`async def post_result(client, channel: str, thread_ts: str, result: dict, is_code_task: bool) -> None`**
Post the final result message to the thread.

`result` dict shape (from Phase 2 agent.py): `{session_id, result, subtype, num_turns, partial_texts}`.
- `subtype` values for errors: `"error_timeout"`, `"error_cancelled"`, `"error_internal"`
- `subtype` for success: `"success"` (or absent/None)

Logic:
1. If `subtype` is an error type → call `_format_error(subtype, result.get("result", ""), result.get("partial_texts", []))`
2. Else → call `_format_completion(result.get("result", ""), is_code_task)`
3. Check for MR URL in result text: `mr_match = MR_URL_RE.search(result.get("result", ""))`
4. If MR found, append `"\n\nMR: {mr_match.group(0)}"` to the message
5. Import and use `formatter.split_long_message(msg)` for chunking
6. For each chunk: `await client.chat_postMessage(channel=channel, thread_ts=thread_ts, text=chunk)`

**Private helpers (same file):**

`def _format_error(subtype: str, result_text: str, partial_texts: list) -> str`:
- `error_timeout` → "Task timed out. Here's what was completed:\n{partial_texts[-1] if partial_texts else '(nothing)'}"
- `error_cancelled` → "Task was cancelled."
- `error_internal` (or unknown) → "Task failed: {result_text[:500] if result_text else 'unknown error'}"

`def _format_completion(result_text: str, is_code_task: bool) -> str`:
- If `is_code_task` is False (Q&A) → return `result_text` directly (the answer is the completion)
- If `is_code_task` is True → return `result_text` (Claude's output includes the diff summary; we don't re-summarize)
- Fallback if empty result_text → "Done." (never return empty string to Slack)
