Create `scripts/setup_glab.sh` — an idempotent shell script to install and authenticate `glab` on the GCP VM. The script must:

1. Check if `glab` is already installed (`command -v glab`) and skip install if so
2. Install via the official packagecloud Debian script:
   ```bash
   curl -s https://packagecloud.io/install/repositories/gitlab/cli/script.deb.sh | sudo bash
   sudo apt install glab -y
   ```
3. Source `.env` from the script directory's parent (the bot root) to load `GITLAB_TOKEN`:
   ```bash
   SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
   BOT_ROOT="$(dirname "$SCRIPT_DIR")"
   set -a; source "$BOT_ROOT/.env"; set +a
   ```
4. Authenticate glab non-interactively (stores to `~/.config/glab-cli/`):
   ```bash
   glab auth login --token "$GITLAB_TOKEN" --hostname gitlab.com
   ```
5. Verify: run `glab auth status` and `glab mr list --repo <repo>` where repo is derived from `GITLAB_REMOTE_URL` env var (add this to .env scaffold if not present). Print success/failure.
6. Add `#!/usr/bin/env bash` shebang and `set -euo pipefail`

The script is meant to be run once on the VM as the bot user after `.env` is populated. Mark it executable (add note in script header: `chmod +x scripts/setup_glab.sh`).

Do NOT hardcode any repo path or token. Read everything from .env.
