```python
def _build_prompt(
    user_text: str,
    worktree_path: str | None,
    channel: str,
    thread_ts: str,
) -> str:
    """Construct the agent prompt with operational context injected."""
    ts_nodot = thread_ts.replace(".", "")
    slack_link = f"https://slack.com/archives/{channel}/p{ts_nodot}"
    lines = [user_text]
    if worktree_path:
        lines += [
            "",
            f"Working directory for this task: {worktree_path}",
            "This is an isolated git worktree. Commit your changes to this worktree's branch.",
            "When creating an MR, target the 'develop' branch.",
            "MR description MUST include all four of the following sections:",
            "  1. What was changed — a brief summary of the change and its purpose",
            "  2. Files changed — a list of every file you created or modified",
            "  3. Test results — the pytest output (or 'No tests run' if tests were not relevant)",
            f"  4. Slack thread link: {slack_link}",
        ]
    return "\n".join(lines)
```

**3. Add `_run_agent_real` async function**:

Follow the research Pattern 1 exactly (from `03-RESEARCH.md`), with these specifics:

- Extract `thread_ts = event.get("thread_ts") or event["ts"]` (use thread_ts for thread reply, fall back to event ts for new messages)
- Extract `channel = event["channel"]`, `text = event.get("text", "")`, `user_id = event.get("user", "")`
- Strip @mention: `clean_text = re.sub(r"<@[A-Z0-9]+>", "", text).strip()` (import `re` if not already)
- Determine `is_code_task_flag = worktree.is_code_task(clean_text)`
- Create `worktree_path_val = None`
- If `is_code_task_flag`: `worktree_path_val = await worktree.create(thread_ts, clean_text)` — wrap in try/except, on failure post error to thread and return
- Get `session_id = session_map.get(channel, thread_ts)` (may return None for new threads)
- Build `prompt = _build_prompt(clean_text, worktree_path_val, channel, thread_ts)`
- Create `on_message_cb = progress.make_on_message(client, channel, thread_ts)`

- Define `async def notify_cb()`: calls `await progress.post_started(client, channel, thread_ts, clean_text)`
- Define `async def result_cb(result: dict)`:
  1. If `result.get("session_id")`: `session_map.set(channel, thread_ts, result["session_id"])`
  2. On failure subtypes (`error_timeout`, `error_cancelled`, `error_internal`): call `await worktree.stash(thread_ts)` before posting result
  3. `await progress.post_result(client, channel, thread_ts, result, is_code_task_flag)`

- Create `QueuedTask`:
  ```python
  task = QueuedTask(
      prompt=prompt,
      session_id=session_id,
      channel=channel,
      thread_ts=thread_ts,
      user_id=user_id,
      cwd=worktree_path_val,
      notify_callback=notify_cb,
      result_callback=result_cb,
      on_message=on_message_cb,
  )
  ```
  Note: `on_message` may not yet be a field on `QueuedTask` — if not, either add it here or pass it through `notify_callback`. The cleanest path: add `on_message: object = None` field to `QueuedTask` in `bot/queue_manager.py` and pass it to `run_agent_with_timeout` in the queue loop. If this field is not yet in `QueuedTask` from Plan 02, add it here by also modifying `bot/queue_manager.py`.

- Enqueue: `if not enqueue(task): await client.chat_postMessage(channel=channel, thread_ts=thread_ts, text=formatter.format_queue_full(queue_depth(), ""))`
