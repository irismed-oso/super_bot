**bot/handlers.py** — all event and command handlers. Structure as a `register(app)` function to avoid circular imports with app.py.

Implement in this exact order within `register()`:

1. **@mention handler (SLCK-02, SLCK-05 — lazy listener pattern):**

```python
def register(app: AsyncApp) -> None:
    @app.event("app_mention", lazy=[_run_agent_stub])
    async def handle_mention_ack(ack, body, client, event):
        # Guard 1: Filter bot's own messages (SLCK-04)
        if is_bot_message(event):
            await ack()
            return

        # Guard 2: Event deduplication (SLCK-06)
        event_id = body.get("event_id", "")
        if event_id and is_seen(event_id):
            await ack()
            return

        # Guard 3: Access control (SLCK-03)
        user_id = event.get("user", "")
        if not is_allowed(user_id):
            await ack()
            return  # Silent ignore — bot appears offline to unauthorized users

        # Guard 4: Channel filter
        channel_id = event.get("channel", "")
        if not is_allowed_channel(channel_id):
            await ack()
            return

        # All guards passed — mark as seen and ack immediately
        if event_id:
            mark_seen(event_id)
        await ack()

        # Immediate acknowledgment to user (within 3-second window)
        await client.reactions_add(
            channel=event["channel"],
            name="hourglass_flowing_sand",
            timestamp=event["ts"]
        )
        thread_ts = event.get("thread_ts") or event["ts"]
        await client.chat_postMessage(
            channel=event["channel"],
            thread_ts=thread_ts,
            text="Working on it."
        )

    async def _run_agent_stub(body, client, event):
        """Phase 1 stub — Phase 2 replaces this with real Claude Agent SDK invocation."""
        import structlog
        log = structlog.get_logger()
        thread_ts = event.get("thread_ts") or event["ts"]
        channel = event["channel"]
        text = event.get("text", "")
        log.info("agent_stub_called", channel=channel, thread_ts=thread_ts, text_preview=text[:80])
        # Record task in state
        await task_state.set_current({"text": text, "user": event.get("user", ""), "ts": event["ts"]})
        # Post stub completion — Phase 2 replaces this
        await client.chat_postMessage(
            channel=channel,
            thread_ts=thread_ts,
            text="[Phase 1 — agent not yet connected. Phase 2 will wire Claude Code here.]"
        )
        await task_state.clear_current()
```

**systemd/superbot.service** (INFRA-04):
Create systemd/ directory and the service unit file. This file is copied to /etc/systemd/system/ by startup.sh.

```ini
[Unit]
Description=SuperBot Slack Bridge
Documentation=https://gitlab.com/irismed/super_bot
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=bot
Group=bot
WorkingDirectory=/home/bot/super_bot
EnvironmentFile=/home/bot/.env
ExecStart=/home/bot/super_bot/.venv/bin/python -m bot.app
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal
SyslogIdentifier=superbot

# Resource limits
LimitNOFILE=65536

# Security hardening
NoNewPrivileges=true
PrivateTmp=true

[Install]
WantedBy=multi-user.target
```

Key decisions:
- `EnvironmentFile=/home/bot/.env` — loads credentials without them appearing in `systemctl show` (use EnvironmentFile, not Environment= inline per research)
- `User=bot` — bot process runs as low-privilege user, not root
- `Restart=always` with `RestartSec=5` — auto-restart on crash with 5-second delay
- `StandardOutput=journal` — logs to journald, accessible via `journalctl -u superbot`
- `NoNewPrivileges=true` + `PrivateTmp=true` — defence-in-depth security hardening
- `WorkingDirectory=/home/bot/super_bot` — bot.app module path resolution requires this

**slack_manifest.yaml**:
The Slack app manifest to create the SuperBot app at api.slack.com/apps. Note: slash commands in Socket Mode do NOT include a `url:` field (pitfall #3 in research — omit the URL entirely).

```yaml
_metadata:
  major_version: 2
  minor_version: 1

display_information:
  name: SuperBot
  description: Autonomous coding assistant for mic_transformer

settings:
  socket_mode_enabled: true
  event_subscriptions:
    bot_events:
      - app_mention

features:
  bot_user:
    display_name: SuperBot
    always_online: false
  slash_commands:
    - command: /status
      description: Show current task, recent history, and uptime
      should_escape: false
    - command: /cancel
      description: Stop the currently running task
      should_escape: false
    - command: /help
      description: Show what SuperBot can do and available commands
      should_escape: false

oauth_config:
  scopes:
    bot:
      - app_mentions:read
      - chat:write
      - reactions:write
      - commands
```
