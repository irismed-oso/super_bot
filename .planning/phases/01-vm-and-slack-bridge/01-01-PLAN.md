Create a Terraform configuration targeting the IrisMed GCP project. Use hashicorp/google provider ~> 5.0.

**terraform/variables.tf** — declare these variables with sensible defaults:
- `gcp_project` (string, required, no default — operator must pass)
- `region` (string, default = "us-west1")
- `zone` (string, default = "us-west1-a")
- `machine_type` (string, default = "e2-small")
- `bot_disk_size_gb` (number, default = 20)

**terraform/main.tf** — create these resources:
1. `google_service_account` "superbot" with account_id = "superbot-sa"
2. `google_compute_instance` "superbot" with:
   - name = "superbot-vm"
   - machine_type = var.machine_type
   - zone = var.zone
   - boot_disk: image family "ubuntu-2404-lts" from project "ubuntu-os-cloud", size = var.bot_disk_size_gb, type = "pd-balanced"
   - network_interface: network = "default", access_config {} (ephemeral external IP)
   - metadata_startup_script = file("${path.module}/startup.sh")
   - service_account: email = google_service_account.superbot.email, scopes = ["cloud-platform"]
   - tags = ["superbot"]
   - metadata = { enable-oslogin = "TRUE" }
3. `google_compute_firewall` "superbot_egress": allow TCP 443 and 80 egress on targets tagged "superbot" to 0.0.0.0/0

Add a comment in main.tf at the top: "# Verify Ubuntu 24.04 image family before applying: gcloud compute images list --filter='name~ubuntu-24' --project=ubuntu-os-cloud"

**terraform/outputs.tf** — output:
- `vm_external_ip` = google_compute_instance.superbot.network_interface[0].access_config[0].nat_ip
- `vm_name` = google_compute_instance.superbot.name
- `service_account_email` = google_service_account.superbot.email

Do NOT include a remote backend or Terraform Cloud configuration — use local state. The operator can migrate later.

<task type="auto">
  <name>Task 2: VM startup script (bootstrap)</name>
  <files>
    terraform/startup.sh
  </files>
  <action>
Create terraform/startup.sh — the one-time bootstrap script that runs on first VM boot as root.

Write the script with `set -euo pipefail` and redirect stdout/stderr to /var/log/startup.log.

**Order of operations (strictly follow this sequence):**

1. System packages: `apt-get update -qq && apt-get install -y -qq git python3 python3-pip curl`

2. Install uv (fast Python package manager):
   ```bash
   curl -LsSf https://astral.sh/uv/install.sh | sh
   export PATH="/root/.cargo/bin:$PATH"
   ```

3. Install Node.js 20.x (required for Claude Code CLI):
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
   apt-get install -y nodejs
   ```

4. Create bot Linux user (no sudo, no login shell beyond bash):
   ```bash
   if ! id bot &>/dev/null; then
       useradd -m -s /bin/bash -c "SuperBot service user" bot
   fi
   ```

5. As the bot user, clone the super_bot repo and set up venv:
   ```bash
   sudo -u bot bash << 'BOT'
       cd /home/bot
       git clone https://gitlab.com/irismed/super_bot.git super_bot
       /root/.cargo/bin/uv venv /home/bot/super_bot/.venv
       source /home/bot/super_bot/.venv/bin/activate
       /root/.cargo/bin/uv pip install -r /home/bot/super_bot/requirements.txt
   BOT
   ```
   Note: mic_transformer clone is deferred — requires GITLAB_TOKEN from .env which operator populates manually. Leave a comment with the command to run after .env is populated.

6. Create placeholder .env file for bot:
   ```bash
   sudo -u bot bash << 'BOTENV'
       cat > /home/bot/.env << 'ENV'
   SLACK_BOT_TOKEN=REPLACE_ME
   SLACK_APP_TOKEN=REPLACE_ME
   GITLAB_TOKEN=REPLACE_ME
   ALLOWED_USERS=REPLACE_ME
   ALLOWED_CHANNEL=REPLACE_ME
   ENV
       chmod 600 /home/bot/.env
   BOTENV
   ```

7. Configure GitLab git credentials store as bot user (INFRA-06):
   ```bash
   sudo -u bot bash << 'GITBOT'
       git config --global credential.helper store
       # Placeholder — operator populates GITLAB_TOKEN in .env, then runs:
       # echo "https://oauth2:${GITLAB_TOKEN}@gitlab.com" > ~/.git-credentials
       # chmod 600 ~/.git-credentials
       touch /home/bot/.git-credentials
       chmod 600 /home/bot/.git-credentials
       chown bot:bot /home/bot/.git-credentials
   GITBOT
   ```

8. Install Claude Code CLI as bot user (INFRA-03):
   ```bash
   sudo -u bot bash -c 'npm install -g @anthropic-ai/claude-code'
   ```
   Add comment: "# After deploy: SSH as bot user and run 'claude login' interactively once"

9. Install systemd service (do NOT start yet — .env must be populated first):
   ```bash
   cp /home/bot/super_bot/systemd/superbot.service /etc/systemd/system/
   systemctl daemon-reload
   systemctl enable superbot
   # Do NOT start: systemctl start superbot
   # Start after .env is populated — see DEPLOY.md
   ```

10. Print completion message to log.

IMPORTANT security requirements:
- /home/bot/.env must be chmod 600, owned by bot:bot
- /home/bot/.git-credentials must be chmod 600, owned by bot:bot
- The bot user must NOT have sudo access
- No secrets in the startup script or in Terraform state — .env files contain only placeholders
