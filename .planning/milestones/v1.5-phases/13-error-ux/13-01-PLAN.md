def get_active_monitors() -> list[dict]:
    """Return list of active background monitors (date_str, run_count, started_at)."""
    return [
        {"date_str": m["date_str"], "run_count": m["run_count"],

        for m in _active_monitors
    ]

**2. Add status handler to bot/fast_commands.py:**
Import `queue_manager` and `background_monitor` at the top of the file (from bot import queue_manager, background_monitor).

Create a regex pattern:
```python
_BOT_STATUS_RE = re.compile(
    r"(?:are\s+you\s+(?:broken|stuck|still\s+(?:going|running|working|there))|"
    r"what\s+(?:are\s+you\s+doing|is\s+(?:your|the)\s+status)|"
    r"you\s+(?:ok|okay|alive|there)\??|"
    r"bot\s+status)",
    re.IGNORECASE,
)
```

Create async handler `_handle_bot_status(text: str, **kwargs) -> str`:
- Call `queue_manager.get_state()` to get current task and queue depth
- Call `background_monitor.get_active_monitors()` to get background tasks
- Build a response:
  - If a task is running: ":gear: *Running a task*\n`{task.clean_text[:100]}`\nQueue: {depth} waiting"
  - If background monitors active: ":satellite: *Background crawl in progress*\nTracking {run_count} locations for {date_str} ({elapsed_s}s elapsed)"
  - If idle and no background: ":white_check_mark: *Idle* -- no tasks running, no background jobs.\nUptime: {uptime}"
- Import `task_state` to get uptime: `from bot import task_state`

**3. Register in FAST_COMMANDS:**
Add `(_BOT_STATUS_RE, _handle_bot_status)` to the FAST_COMMANDS list. Place it AFTER the existing entries (status queries about eyemed should still match the eyemed status handler first, not this generic status handler).
