       - Add `_LOCATION_EXTRACT_RE` that matches known location names directly from the text. Sort by length descending so "optical image" matches before "optical".

       - Fall back to _LOCATION_RE if no known location matched.

    3. Update _EYEMED_STATUS_RE to also match "status on" pattern:
       - Add `r"|status\s+on\s+\S+\s+eyemed"` to the alternation so "status on DME eyemed" is caught.

    4. Handle "today" in date parsing: if the text contains "to today" or "to now", compute today's date as MM.DD.YY and use it as --to. Currently "today" is not converted to a date — only _DATE_RE captures numeric dates.
