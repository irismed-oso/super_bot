  <action>
Create bot/git_activity.py with a single public async function:

async def capture_git_activity(
    result: dict,
    cwd: str | None,
    channel: str,
    thread_ts: str,
) -> list[dict]:

This function runs AFTER the agent session completes and captures git activity:

1. **Commit capture (GITLOG-01 + GITLOG-03):**
   - If cwd is None (no worktree), skip commit capture (read-only task).
   - Otherwise, run `git log --format="%H|%s|%D" --name-only` in the worktree cwd to get commits on the current branch that are not on origin/develop (use `origin/develop..HEAD`). If origin/develop is not available, fall back to `HEAD~10..HEAD` with a dedup guard.
   - Parse each commit into: hash (full SHA), message (subject line only -- first line), branch (from %D or from `git rev-parse --abbrev-ref HEAD`), repo name (derived from cwd basename -- e.g. "/home/bot/mic_transformer" -> "mic_transformer", worktree paths resolve via realpath to find the main repo name).
   - For each commit, collect the list of changed file paths (from --name-only output, file names only, no diff stats per CONTEXT.md decision).
   - Repo name derivation: use `git rev-parse --show-toplevel` in cwd, then take the basename. This handles worktrees correctly (resolves to main repo).
   - Each commit becomes an activity_log.append() call with type="git_commit":
     ```
     {"type": "git_commit", "hash": "abc123...", "message": "fix timeout bug",
      "repo": "mic_transformer", "branch": "superbot/fix-timeout",
      "files": ["bot/agent.py", "bot/progress.py"],
      "channel": channel, "thread_ts": thread_ts, "ts": <iso timestamp>}
     ```

2. **PR capture (GITLOG-02):**
   - Import PR_URL_RE from bot.progress (already exists).
   - Search result.get("result", "") for PR URLs using PR_URL_RE.findall().
   - For each URL found, extract a title: check result text for the line containing the URL and grab adjacent text, or use the URL path as fallback title (e.g. "pull/42" or "merge_requests/42").
   - Derive repo name from the URL path (e.g. "github.com/org/repo/pull/42" -> "repo").
   - Each PR becomes an activity_log.append() call with type="git_pr":
     ```
     {"type": "git_pr", "url": "https://...", "title": "Fix timeout bug",
      "repo": "mic_transformer",
      "channel": channel, "thread_ts": thread_ts, "ts": <iso timestamp>}
     ```

3. **Return value:** Return the list of all entries appended (for logging/testing convenience).

4. **Error handling:** Wrap git subprocess calls in try/except. If git commands fail (e.g., not a git repo, worktree already removed), log a warning via structlog and return empty list. Never let git activity capture crash the main result flow.

5. **Deduplication:** To avoid logging the same commits on follow-up messages in the same thread, track which commit hashes have already been logged. Read existing entries for today from activity_log.read_day(date.today()), filter to type="git_commit" with matching thread_ts, and skip any hash already present.

Use asyncio.create_subprocess_exec for git commands (same pattern as worktree.py). Use structlog for logging. Import activity_log for persistence.

```python
def read_day_by_type(d: date, entry_type: str) -> list[dict]:
    """Read activity entries for a given date filtered by type field."""
    return [e for e in read_day(d) if e.get("type") == entry_type]
```
