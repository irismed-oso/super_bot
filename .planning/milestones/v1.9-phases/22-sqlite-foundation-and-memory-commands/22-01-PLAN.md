**Schema (single `memories` table + FTS5 virtual table):**

```sql
CREATE TABLE IF NOT EXISTS memories (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    category    TEXT NOT NULL CHECK (category IN ('rule', 'fact', 'history', 'preference')),
    content     TEXT NOT NULL,
    source_user TEXT NOT NULL,
    source_channel TEXT,
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    active      INTEGER NOT NULL DEFAULT 1
);

CREATE VIRTUAL TABLE IF NOT EXISTS memories_fts USING fts5(
    content,
    content='memories',
    content_rowid='id',
    tokenize='porter unicode61'
);

-- Triggers to keep FTS in sync
CREATE TRIGGER IF NOT EXISTS memories_ai AFTER INSERT ON memories BEGIN
    INSERT INTO memories_fts(rowid, content) VALUES (new.id, new.content);
END;
CREATE TRIGGER IF NOT EXISTS memories_ad AFTER DELETE ON memories BEGIN
    INSERT INTO memories_fts(memories_fts, rowid, content) VALUES('delete', old.id, old.content);
END;
CREATE TRIGGER IF NOT EXISTS memories_au AFTER UPDATE ON memories BEGIN
    INSERT INTO memories_fts(memories_fts, rowid, content) VALUES('delete', old.id, old.content);
    INSERT INTO memories_fts(rowid, content) VALUES (new.id, new.content);
END;
```

**Connection management:**
- Module-level `_conn: aiosqlite.Connection | None = None`
- `async def init(db_path: str | None = None)` -- opens connection, enables WAL mode (`PRAGMA journal_mode=WAL`), sets `PRAGMA busy_timeout=5000`, creates schema. Default path: `/home/bot/data/superbot_memory.db` (configurable via `MEMORY_DB_PATH` env var in config.py, fallback to `./data/superbot_memory.db` for local dev)
- `async def close()` -- closes connection, sets `_conn = None`
- All operations check `if _conn is None: return None/[]` at the top (graceful degradation)

**CRUD functions:**
- `async def store(content: str, category: str, source_user: str, source_channel: str | None = None) -> int | None` -- INSERT, returns row id
- `async def search(query: str, limit: int = 10, category: str | None = None) -> list[dict]` -- FTS5 search with BM25 ranking. Query: `SELECT m.id, m.category, m.content, m.source_user, m.created_at, rank FROM memories m JOIN memories_fts ON m.id = memories_fts.rowid WHERE memories_fts MATCH ? AND m.active = 1 {AND m.category = ?} ORDER BY rank LIMIT ?`. Returns list of dicts with keys: id, category, content, source_user, created_at, rank.
- `async def deactivate(memory_id: int) -> bool` -- SET active = 0 (soft delete). Returns True if a row was affected.
- `async def list_all(category: str | None = None, limit: int = 50) -> list[dict]` -- SELECT all active memories, optionally filtered by category, ordered by created_at DESC. Returns list of dicts.
- `async def get_by_id(memory_id: int) -> dict | None` -- SELECT single memory by id.

**Auto-categorization helper:**
- `def categorize(content: str) -> str` -- Simple keyword heuristic for CMD-01 auto-categorization:
  - If content starts with "always", "never", "rule:", "when...always", "when...never" -> "rule"
  - If content starts with "I prefer", "use...instead", "preference:" -> "preference"
  - Default -> "fact"
  - (history category is reserved for auto-capture in Phase 24)

Add `aiosqlite>=0.21.0` to requirements.txt (append to existing file, do not overwrite other deps).

**Important:** Use `await _conn.execute()` and `await _conn.commit()` for writes. Use `await _conn.execute_fetchall()` for reads. Wrap FTS5 MATCH queries in try/except to handle malformed queries (FTS5 syntax errors) -- on error, fall back to a simple LIKE query.
