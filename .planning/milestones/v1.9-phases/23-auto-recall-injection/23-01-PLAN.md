<task type="auto">
  <name>Task 1: Create bot/memory_recall.py with recall block builder</name>
  <files>bot/memory_recall.py</files>
  <action>
Create `bot/memory_recall.py` with an async function `build_recall_block(user_text: str) -> str | None` that:

1. **Always fetch all rules**: Call `memory_store.list_all(category="rule")` to get all active rule memories. Rules are non-negotiable institutional knowledge and are always included regardless of query.

2. **FTS5 search for remaining relevant memories**: Call `memory_store.search(user_text, limit=10)` to get BM25-ranked results from the user's message text. Filter out any results that are already in the rules set (avoid duplicates). Filter out memories with category "history" (task history is low-value noise for agent prompts).

3. **Cap at 5-8 total memories**: All rules are included first. Fill remaining slots (up to 8 total) with the top FTS5 results by rank. If rules alone exceed 8, include all rules anyway (rules are non-negotiable).

4. **Token budget guard**: Calculate approximate token count (len(content) // 4 as rough estimate). If total exceeds 500 tokens, truncate from the bottom of the FTS5 results (keep all rules). Log a warning if truncation occurs.

5. **Format as a prompt block**: Return a formatted string block that will be injected into the agent prompt. Format:

```
RECALLED MEMORIES (from team knowledge base):
- [rule] Always run with dry_run=True first before any production posting
- [rule] VSP files use 835 format, EyeMed uses PDF
- [fact] The DME location code is "DME-001" in the Crystal PM system
- [preference] Nicole prefers summary tables over long text output

(Remembered: {count} memories applied)
```

Each line is prefixed with the category in brackets. The footer "(Remembered: N memories applied)" serves as the citation line (RECALL-03).

6. **Return None if no memories found** (no rules exist and FTS5 returns nothing). Do NOT inject an empty block.

7. **Graceful degradation**: Wrap the entire function in try/except. If memory_store is unavailable (connection is None, DB error), log a warning and return None. Never crash the agent pipeline due to recall failure.

Use structlog for logging. Follow the same module patterns as `bot/memory_store.py` (module-level functions, structlog logger at top).
